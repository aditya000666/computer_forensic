Describe about Linux Directory Structure


/bin


• Place for most commonly used terminal commands
• Example, ls, mount, rm etc.

/boot


• Contains files needed to start up file system
• Including Linux Kernel, RAM disk image and boot loader configuration files

/dev


• Contains all device files which are not regular files
• Instead refer to various hardware devices on system

/etc


• Contain system global configuration files
• Affect system behaviour for all users

/home


• Place for user home directories
• Mostly include
• Documents
• Pictures

/lib


• Contains important dynamic libraries and kernel modules

/media


• Contains mount point for external media
• Example hard drives or removable media

/mnt


• Place for mount points
• Dedicated to temporary mounted device
• For example, mounting directory from a network

/opt


• Store additional software for system
• Not handled by the package manager

/proc


• Virtual filesystem provides mechanism for kernel to send information to process

/root


• Superuser home directory

/run


• Temporary file system available in boot process where run-time data is stored
• Files under this are removed or truncated at beginning of boot process

/sbin


• Contains important administrative command
• Generally employed by superuser

/srv


• Contain data directories of services
• For example HTML or FTP

/sys


• Virtual filesystem accessed to set or obtain information about kernel view of system

/tmp


• Temporary files stored by applications

/usr


• Contains majority of user utilities and application
• Partly replicates root directory structure
• For instance /usr/bin and /usr/lib

/var


• Variable data
• Like Logs, databases, websites, temporary spool (email)
• Notable directory /var/log
