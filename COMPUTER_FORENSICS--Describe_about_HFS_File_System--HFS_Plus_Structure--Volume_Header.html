Volume Header


• Each HFS Plus Volume contains Volume Header 1024 bytes from start of volume
• Volume Header analogous to Master Directory Block (MDB) for HFS containing information about volume as whole which includes location of other key structure in volume
• Copy of Volume Header, Alternate Volume Header is stored starting 1024 bytes before the end of volume
• Implementation should update this copy when length or location of one special file changes
• Alternate Volume Header is intended for disk repair utilities
• First 1024 bytes and Last 512 bytes of volume header are reserved
• First 1024 bytes are used for boot block
• Traditional Mac OS Finder will write to System Folder Changes
• Last 512 bytes were used during Apple CPU manufacturing process
• Alternate Volume Header is always stored at offset 1024 bytes from end of volume
• If disk size is not even multiple of allocation of block size the area may lie beyond last allocation block
• Last allocation block is still reserved if Alternate Volume Header is not stored
• Allocation block containing first 1536 bytes (1024 Volume Header + 512 Reserved Space) are marked as used in Allocation File
• To accommodate AVH and reserved space, the last allocation block is marked as used in allocation file
• Volume Header is described by HFS Plus Volume Header type
