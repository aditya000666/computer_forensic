MFT


• MFT is a file
• MFT has entry of itself
• First entry in table is named $MFT and describes on-disk location of MFT
• Starting location of MFT is given in boot sector which is located in first sector of file system
• Microsoft does not delete MFT entries after they have been created
• Size of MFT is defined in boot sector (1024 bytes)
• First 42 bytes contain 12 fields and remaining 982 bytes are unstructured and filled with attributes
• First field in MFT is signature and standard entry will have ASCII
• If error found in entry the string entry is “BAD”
• There is flag field that identifies if entry is being used and if entry is for directory
• If file cannot fit attributes in one entry, use multiple entries
• When this occurs, first entry is called Base File Record or Base MFT Entry
• Nearly every allocated MFT entry has $FILENAME and $STANDARD_INFORMATION
• $FILENAME contains filename, size and temporal information
• $STANDARD_INFORMATION attribute contains temporal, ownership and security information
• Every file has $DATA attribute which contains file content. If content is 700 bytes, it becomes nonresident and saved in external cluster
• File with more than one $DATA attribute, additional attributes are referred to as Alternate Data Streams (ADS)
• NTFS provides capability for attribute contents to be encrypted
• When attribute encrypted, only content is encrypted and attribute header is not
• $LOGGED_UTILITY_STREAM attribute is created for file and contain keys needed to decrypt the data
