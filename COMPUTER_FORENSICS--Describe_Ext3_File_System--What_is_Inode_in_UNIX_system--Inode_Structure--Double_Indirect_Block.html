Double Indirect Block


• If size of file is greater than 4MB + 48K, then inode will start using Double Indirect Block pointer to address data blocks.
• Double Indirect block will point to block that comes after 4MB + 48K which will point to blocks where data is stored
• Double indirect pointer can address 1024 Indirect block pointers (1024 * 4M=4TB) of file size
• The 15th block pointer in inode will point to block just after 4G of data which will point to 1024 Double Indirect Block Pointer
